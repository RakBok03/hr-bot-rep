# Обзор проекта

HR Task Bot — система для заявок на подбор персонала.

## Состав

- **Telegram bot (`bot/`)**
  - уведомления и инлайн‑ответы;
  - ежедневные напоминания:
    - обратная связь по собеседованию;
    - заявки с прошедшей “Датой выхода”.
- **WebApp (`webApp/`)**
  - API на FastAPI;
  - статический фронт (Telegram WebApp).
- **Синхронизация (`sheet_sync/`)**
  - читает Google Sheet;
  - создаёт/обновляет кандидатов в БД;
  - уведомляет автора заявки в Telegram.
- **База (`db/`)**
  - модели и репозитории (по умолчанию SQLite; в Docker общий том `bot_data:/app/data`).
- **Интеграции (`integrations/`)**
  - клиент записи/обновления строк в Google Sheets.

## Потоки данных (кратко)

- **Создание/редактирование заявки**: WebApp → DB → Google Sheets → уведомление HR в чат.
- **Добавление кандидата из таблицы**: Google Sheets → sheet_sync → DB → уведомление владельцу заявки (Telegram).
- **Напоминания**: bot планировщик → DB → сообщения владельцам (Telegram).
